# キャリアパス

## はじめに

本資料はキャリアパスについてまとめた資料です。  
筆者の経験からの内容になり、企業・案件によって役割が異なります。  
開発手法についてはウォーターフォール前提とします。  

## 経験年数

### テスト

基本的にはテスト経験が少しでもあれば案件はあります。  
ですが、テストエンジニアは人数が多いため、テスト・開発経験が長い人が採用されるケースが多く  
できるだけ長いIT経験が必要になります。

上流を目指していくと、「JSTQB Foundation Level」資格が必要になったり  
プログラミング（テストコードが書けるレベル）が必要になってきます。  

- 求められるスキルの例
  - 「IT業界での実務経験」、「IT経験が３年以上ある方(目安)」
  - 「テスト実施経験」
  - 「テスト経験半年以上」
  - 「テスト設計経験（テストケース作成、テスト項目書作成等）」
  - 「シナリオ作成経験」
  - 「単体～システムテストまで経験している方」
  - 「Webアプリケーションの品質保証業務経験1年以上」
  - 「テストエンジニアとしての実務経験3年以上」
  - 「テストケース作成経験3年以上」
  - 「SQLを読めること（JOIN、副問い合わせくらい）」
  - 「JSTQB テスト技術者資格 Foundation Level の知見」
- 尚可スキルの例
  - 開発経験
  - DB構築経験
  - テストコード作成経験
  - 業務効率化経験
  - JSTQB テスト技術者資格 Foundation Level の知見

### 開発

開発経験年数としては、3年・5年の経験を求める企業・案件が多いため  
選択の幅を広げるため、まずは **開発経験3年** を目標とするのが良いです。  

- 求められるスキルの例
  - 「何らかのシステム開発経験　実務３年以上」
  - 「何らかのシステム開発経験　実務５年以上」
  - 「Web開発（サーバサイドエンジニア）　実務３年以上」

上記に加え、言語・フレームワークの経験は1年,2年以上としている所が多そうです。  
（2023/10現在 PHP経験の最低ラインが3年になっている募集が多くなってきました）

- 求められる言語スキルの例
  - 「PHP, Go言語　のいずれか　実務２年以上」
  - 「C＃, Java　のいずれか　実務１年以上」
  - 「Laravel 実務１年以上」

## PMO(プロジェクトマネジメントオフィサー)

SES案件的にはPSOに分類される  
SE・PMの中間に入る職種  
小規模なプロジェクトには設けられない事が多いが  
大きいプロジェクトの場合に設けられる  

必要なスキルや業務内容については、案件によって異なり  
資料作成などの雑用業務のケースがあったり、PLのように業務全般を任されることがある  

- あった方が良いスキル
  - Excel（グラフ作成、集計等）
  - ExcelVBA（マクロ）

### 下流

- 必要なスキル
  - コミュニケーション力
  - 基本的なPC操作ができればOK
- 任される業務
  - 議事録作成
  - 文書作成
  - 雑務

### 上流

上流のPMOになってくると、要求定義・要件定義・基本設計等のウォーターフォールの上流工程や  
PMの一部業務をやるようになります。  

- 必要なスキル
  - コミュニケーション力
  - システム知識
  - ドキュメンテーションスキル
- 任される業務
  - プロジェクト管理（進捗管理・課題管理等）
  - 議事録の作成、討議資料の作成
  - 業務フロー図、データフロー図などの各種フロー図の作成

## インフラエンジニア

責任のある仕事のため、SESではなりにくい、職種です。  
近いものだと保守・運用になります。  
将来的にインフラエンジニアを目指す場合、SE・保守・運用の経験を積みつつ  
様々な資格を取得することをおすすめします。  

インフラエンジニアと言っても、サーバー・ネットワーク・セキュリティと幅広いです。

- 必要なスキル
  - PG・SEに求められるスキル
  - コミュニケーション力
- 任される業務（様々な求人情報から抜粋）
  - セキュリティ施策の企画・推進
  - セキュリティ啓蒙、教育
  - セキュリティ監視システム開発、運用
  - 不正行為の調査、研究、対策案の検討
  - 最先端技術に関するセキュリティ上の脅威の調査、対策の検討
  - IoT機器・アプリケーションのセキュリティ検査、ペネトレーションテスト
  - マルウェア解析
  - バックドア検査
  - プロフェッショナル向けトレーニング講師 （疑似マルウエア作成、ペネトレーション）
  - セキュリティ関係のツール、コンポーネントの研究試作・請負開発
  - 脆弱性情報データベースの確認
- 資格
  - 基本情報技術者試験
  - 応用情報技術者試験
  - CCNA
  - CCNP
  - LPIC
  - LinuC
  - AWS 認定ソリューションアーキテクト
  - CISSP
- ツール
  - OWASP-ZAP
  - Postman
  - Jmeter

## キャリアパスの例

SESでは任される業務に限りはありますが  
チームで現場参画し、PLとなるケースや  
現場の活躍次第で、PLに抜擢されることがあります。  

- 1年目: PGとして開発経験を積む
- 4年目: SEとして上流工程経験を積む
- 6年目: PLまたは、PMOとしてマネジメント経験を積む
- 8年目: PMまたは、技術的なポジション

## アジャイル開発の場合

一般的に「アジャイル開発」は「スクラム開発」を指すことが多く  
全体的に高いレベルが求めれます。  
とはいえ開発メンバーが得意とする業務を加味してくれる（はず）ので  
経験の浅いエンジニアでも出来ないということはありません。  

- 必要なスキル（優先順）
  - コミュニケーション（必須）
  - 能動的に動ける
  - 提案力
  - 設計
  - プログラミング
  - リーダーシップ（状況によって機能単位で中心となるメンバーを切り替えて進めていく）

筆者の経験上、経験5年以上のエンジニア、個人でやってきたエンジニア  
リーダー経験があるエンジニアで構成されることが多いです。  

```txt
■注意
アジャイル開発を謳い、スクラム開発などを詳しく知らない現場では
プロジェクト管理が杜撰な可能性が高いです。
```
